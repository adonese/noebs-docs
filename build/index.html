Introduction

Noebs is a middleware that consumes EBS services. It was built to be simple, developer friendly and customizable enough to be used in different scenarios and adapt to variety of different logics.

Overview

We make great use of http status codes; it is one of our design decisions. We believe that they make client codes more easy and it helps us too to make a robust API. You only get 200 on successful transactions. You get 400 in case of missing a field, or using wron types You get 502 in EBS payment gateway errors (e.g., insufficient fund)

Authentication

What is the preferred way of using the API?

Error Codes

We use different kind of error codes:
  • 400 for validations errors (e.g., missing a required field)
  • 502 for EBS (payment gateway) errors (e.g., when you have an insufficient balance)
  • 504 for EBS (payment gateway) timeouts. E.g., EBS server is unreachable

Working Key API

POST https://beta.soluspay.net/api/workingKey

Working key is the key used by EBS for card holder's info encryption

Request Body

The request body should be "application/json" encoded, and should contain an object with the following attributes.

Parameter Description
clientId required string Client ID a unique value provided by EBS
systemTraceAuditNumber required Auto increment integer. Unique for each transaction (per day)
tranDateTime required Transaction time in: YYMMddhhmmss format
terminalId required Merchant's terminal ID 
var url = new URL("https://beta.soluspay.net/api/workingKey");
var data = {
    clientId: ...,
    systemTraceAuditNumber: ...,
    tranDateTime: ...,
    terminalId: ...
};
var options = {
    method: "POST",
    body: JSON.stringify(data),
    headers: {
        "content-type": "application/json"
    }
};

fetch(url, options).then(function(response) {
    if (!response.ok) {
        throw new Error(response.statusText);
    }
    console.log(response.json());
})

How to Encrypt your data

NOTE

Please restrain from using the old EBS encryption method. It was insecure and it exposes the plain PIN. (Pre MCS WebService Description 2.8.1)

You are highly advised to use Our online PIN block calculator to make sure your code works!

About the encryption

  • Each client have a unique TMS (Terminal Master Key)
  • /workingKey returns an encrypted key
  • Client use their TMS to decrypt the working key
  • Compute the clear PIN Block as following: p1 = [0] + [pin_length] + [pin] + [10*f] p2 = [0000] + last_12_digits_of_pan_length-1 clear PIN Block = p1 XOR p2
  • Use the decrypted working key as the key for DES
  • Use DES to encrypt the clear PIN Block
